Concrete Syntax


Over the next several weeks, I'm going to try and create my own customized programming language based on 'Clite' from the textbook Programming Language Design. I will tentatively call the language 'Hydra.' It will be a mishmash of imperative and functional concepts, allowing the coder to use concepts from either, and will use a pure object oriented, table-driven interpreter written in Java. It will not have any advanced data structure features such as classes or objects, but will have functions, nested functions, and data types. If I get encouragement, I will take it further.


Some features:

  • A program is simply any number of statements.

  • Declarations and functions are statements

  • A function can be purely functional, where it is just an expression, using an equals sign, and no return is required, or it can be more imperative using a block and a return.

  • Nested functions are allowed, either using the 'where' below a pure function or by defining them in a statement.

  • The type syntax is similar to Haskell and real mathematics. Colon indicates a declaration of types, and an arrow indicates a mapping from one set to another. These type declarations are required for functions.

  • New data types can be declared in a way that is similar to normal declarations. You simply make a constructor with parameters and then specify its type with any identifier. You can make as many constructors as desired, and the types may be recursive.

  • Theoretically I should be able to do partial application, depending on how the semantics is implemented in Java.


Here is the concrete syntax:


Program -> { Statement }


Function -> Identifier : Parameters -> Type ( Block | = Expression [ LocalFs ] )

LocalFs -> where Function { Function }

Parameters -> [ (Parameter) { -> (Parameter) } ]

Parameter -> Identifier : Type

Block -> { { Statement } }


Statement ->

; | \n | Block | Declaration | Assignment | Conditional | Loop | Function

Assignment -> Identifier := Expression ( ; | \n )

Conditional -> if (Expression) Statement [ else Statement ]

Loop -> while (Expression) Statement

Expression -> Term { BinOp Term }

Op -> + | - | * | / | % | **

Term -> [ UnOp ] Primary

UnOp -> - | !

Primary -> ( Expression ) | Type ( Expression ) | Literal | Accessor


Declaration -> Identifier { (Parameter) } : Type


Type -> Int | Bool | Float | Char

Identifier -> Letter { Letter | Digit }

Accessor -> Identifier . Identifier

Letter -> a | b | ... | z | A | B | ... | Z

Digit -> 0 | 1 | ... | 9

Literal -> Integer | Boolean | Floating | Character

Integer -> Digit { Digit }

Boolean -> True | False

Floating -> Integer . Integer

Chararacter -> ' ASCIIChar '
